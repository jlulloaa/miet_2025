Project Brief

Proyecto: Funcionalidad de Autenticación para Nuevos Usuarios

Resumen y Justificación del Proyecto

El presente proyecto tiene como objetivo el desarrollo e implementación de un sistema de autenticación para nuevos usuarios dentro de la aplicación web de monitoreo de pacientes en procesos de kinesiterapia. Actualmente, la aplicación carece de una funcionalidad robusta para la gestión de usuarios, lo cual limita su escalabilidad, dificulta la seguridad y restringe su uso en ambientes clínicos multiusuario. La autenticación permitirá controlar el acceso, garantizar la confidencialidad de los datos clínicos, y preparar la plataforma para una futura integración con instituciones de salud.

Objetivos (SMART)

  • Específico: Desarrollar un módulo de autenticación basado en correo electrónico y contraseña para nuevos usuarios.
  • Medible: El sistema debe permitir el registro y autenticación segura de al menos 100 usuarios únicos.
  • Alcanzable: Se utilizarán tecnologías y marcos de trabajo previamente integrados en la aplicación actual (Node.js, Firebase, React).
  • Relevante: El acceso autenticado es una condición crítica para cumplir con normativas de privacidad y escalabilidad clínica.
  • Temporal: El sistema será desarrollado y probado en un plazo máximo de 5 semanas.

Tiempo del Proyecto

Fecha de inicio: 5 de agosto de 2025
Fecha de término: 9 de septiembre de 2025

Alcance (Scope)

  • Desarrollo del sistema de autenticación (registro, login, logout).
  • Integración con base de datos de usuarios y asignación de roles básicos (paciente, kinesiólogo).
  • Implementación de validaciones de seguridad (contraseñas fuertes, validación de email).
  • Interfaz de usuario para registro e ingreso de usuarios.
  • Documentación técnica del módulo.

Entregables (Deliverables)

  • Módulo backend de autenticación completamente funcional.
  • Interfaz de usuario (UI) para login y registro.
  • Base de datos de usuarios con estructura validada.
  • Manual de usuario para administradores y usuarios finales.
  • Informe final de validación y pruebas funcionales.

Barreras (Constraints)

  • Dependencia de infraestructura de backend ya existente.
  • Limitaciones presupuestarias para contratación de servicios externos.
  • Requisitos normativos de seguridad y confidencialidad de datos clínicos.
  • Disponibilidad limitada de usuarios finales para pruebas piloto.

Recursos Requeridos

  • Recursos Humanos:
    • 1 Desarrollador Full Stack (5 semanas)
    • 1 Diseñador UX/UI (1 semana parcial)
    • 1 Coordinador de pruebas con usuarios (2 semanas)
  • Recursos Financieros:
    • Honorarios equipo técnico: USD $4.500
    • Licencias y herramientas: USD $500
    • Total estimado: USD $5.000
